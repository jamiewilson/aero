---
title: 'Aero Config'
description: 'Use aero.config.ts with defineConfig() to manage Aero options in a dedicated file.'
---

As your project grows, mixing Aero options into `vite.config.ts` alongside Vite-specific settings becomes cluttered. `@aero-js/core/config` lets you move all Aero options into a dedicated `aero.config.ts` with typed config via `defineConfig()`, a clean separation from Vite settings, and support for env-aware configuration.

## Install

<Tabs>
	<Tab title="pnpm">```bash pnpm add @aero-js/core ```</Tab>
	<Tab title="npm">```bash npm install @aero-js/core ```</Tab>
	<Tab title="yarn">```bash yarn add @aero-js/core ```</Tab>
</Tabs>

Import `defineConfig` from `@aero-js/core/config` and `createViteConfig` from `@aero-js/core/vite-config` — both are subpaths of `@aero-js/core`, not separate npm packages.

## Set up

<Steps>
  <Step title="Create aero.config.ts">
    Define your Aero options using `defineConfig()` from `@aero-js/core/config`. You can also override Vite settings via the `vite` key:

    ```ts
    // aero.config.ts
    import { defineConfig } from '@aero-js/core/config'

    export default defineConfig({
      site: { url: 'https://example.com' },
      redirects: [{ from: '/home', to: '/', status: 301 }],
      content: true,
      server: true,
      // Override Vite defaults
      vite: {
        build: {
          minify: false,
        },
      },
    })
    ```

  </Step>
  <Step title="Update vite.config.ts">
    Import your config and pass it to `createViteConfig()` from `@aero-js/core/vite-config`:

    ```ts
    // vite.config.ts
    import { createViteConfig } from '@aero-js/core/vite-config'
    import aeroConfig from './aero.config.ts'

    export default createViteConfig(aeroConfig)
    ```

  </Step>
</Steps>

## Auto-load pattern

If you call `createViteConfig()` with no arguments, it automatically loads `aero.config.ts` from your project root. This avoids the explicit import:

```ts
// vite.config.ts
import { createViteConfig } from '@aero-js/core/vite-config'

export default createViteConfig()
```

<Tip>
	The no-argument form is the recommended pattern for projects scaffolded with `@aero-js/create`. It
	keeps `vite.config.ts` as minimal as possible.
</Tip>

## Env-aware config

Pass a function instead of an object to `defineConfig()` to vary options between dev and production builds. The function receives Vite's `env` object:

```ts
// aero.config.ts
import { defineConfig } from '@aero-js/core/config'

export default defineConfig(env => ({
	site: env.mode === 'production' ? 'https://example.com' : 'http://localhost:5173',
	content: true,
	server: true,
	vite: {
		build: {
			minify: env.mode === 'production',
		},
	},
}))
```

<Note>
	The `env` parameter is the same object Vite passes to a config function — it includes `mode`,
	`command`, and `isSsrBuild`. See the [Vite docs](https://vitejs.dev/config/#conditional-config)
	for details.
</Note>

## Options

`AeroConfig` extends the same plugin options as `aero()` (`AeroOptions`). Use the same fields in `aero.config.ts` that you would pass to `aero()` in `vite.config.ts`.

| Option         | Type                           | Description                                                                                                                  |
| -------------- | ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| `content`      | `boolean \| object`            | Enable content collections. Default `false`. Pass `true` or an options object for `@aero-js/content`.                        |
| `server`       | `boolean`                      | Enable Nitro for API routes and an optional server. Default `false`.                                                         |
| `reactivity`   | `boolean`                      | Enable the Aero reactivity pipeline. Default `false`.                                                                       |
| `hypermedia`   | `boolean`                      | Enable the Aero hypermedia pipeline. Default `false`.                                                                        |
| `site`         | `{ url: string }`              | Canonical site URL for `import.meta.env.SITE` and `Aero.site.url`.                                                           |
| `redirects`    | `Array<{ from, to, status? }>` | Path redirects in dev and Nitro-backed production.                                                                           |
| `middleware`   | `Array`                        | Dev-only request handlers for rewrites and custom responses.                                                                 |
| `dirs`         | `object`                       | Override `client`, `server`, and `dist` directory names.                                                                     |
| `apiPrefix`    | `string`                       | URL prefix for API routes. Default `'/api'`.                                                                                   |
| `vite`         | `UserConfig`                   | **Config file only.** Vite settings merged with Aero defaults.                                                               |
| `incremental`  | `boolean`                      | **Config file only.** When `true` during `vite build`, sets `AERO_INCREMENTAL=1` if unset. See [Incremental builds](/tooling/incremental-builds). |
