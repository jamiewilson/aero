Aero is a static site generator and optional full-stack framework built around plain HTML. You write `.html` files, Aero compiles them at build time into static output, and you deploy anywhere — no runtime, no hydration, no framework overhead.

## How Aero works

Aero compiles your `.html` template files at build time. Build scripts run in Node and are stripped from the output — the browser only receives static HTML, bundled CSS, and the client scripts you explicitly ship.

<Steps>
  <Step title="Scaffold a project">
    Run `pnpm create @aero-js my-app` to generate a new project from the minimal or fullstack template.
  </Step>
  <Step title="Build pages in HTML">
    Create `.html` files in `client/pages/`. Use `{ }` expressions, `<script is:build>`, components, and layouts — all plain HTML syntax.
  </Step>
  <Step title="Run the dev server">
    Run `pnpm dev` to start a Vite dev server with HMR for HTML, CSS, scripts, and content.
  </Step>
  <Step title="Build and deploy">
    Run `pnpm build` to generate a static `dist/` folder. Deploy to any static host, or enable Nitro for API routes and a full server.
  </Step>
</Steps>

<Note>
	Aero is static-first. You get a plain HTML output with no framework runtime in the browser. Enable
	`server: true` only when you need API routes or a dynamic server.
</Note>
