---
title: 'Aero Config'
description: 'Use aero.config.ts with defineConfig() to manage Aero options in a dedicated file.'
---

As your project grows, mixing Aero options into `vite.config.ts` alongside Vite-specific settings becomes cluttered. `@aero-js/core/config` lets you move all Aero options into a dedicated `aero.config.ts` with typed config via `defineConfig()`, a clean separation from Vite settings, and support for env-aware configuration.

## Install

<CodeGroup>
```bash pnpm
pnpm add @aero-js/core
```

```bash npm
npm install @aero-js/core
```

```bash yarn
yarn add @aero-js/core
```

</CodeGroup>

Import `defineConfig` from `@aero-js/core/config` and `createViteConfig` from `@aero-js/core/vite-config` — subpaths of `@aero-js/core`.

## Set up

<Steps>
  <Step title="Create aero.config.ts">
    Define your Aero options using `defineConfig()` from `@aero-js/core/config`:

    ```ts aero.config.ts
    import { defineConfig } from '@aero-js/core/config'

    export default defineConfig({
      site: { url: 'https://example.com' },
      redirects: [{ from: '/home', to: '/', status: 301 }],
      content: true,
      server: true,
    })
    ```

  </Step>
  <Step title="Update vite.config.ts">
    Import your config and pass it to `createViteConfig()` from `@aero-js/core/vite-config`. Put Vite-specific settings in `vite.config.ts`:

    ```ts vite.config.ts
    import { createViteConfig } from '@aero-js/core/vite-config'
    import aeroConfig from './aero.config.ts'
    import { mergeConfig } from 'vite'

    export default mergeConfig(createViteConfig(aeroConfig), {
      build: { minify: false },
    })
    ```

  </Step>
</Steps>

## Auto-load pattern

If you call `createViteConfig()` with no arguments, it automatically loads `aero.config.ts` from your project root. This avoids the explicit import:

```ts vite.config.ts
import { createViteConfig } from '@aero-js/core/vite-config'

export default createViteConfig()
```

<Tip>
	The no-argument form is the recommended pattern for projects scaffolded with `@aero-js/create`. It
	keeps `vite.config.ts` as minimal as possible.
</Tip>

## Env-aware config

Pass a function instead of an object to `defineConfig()` to vary options between dev and production builds. The function receives Vite's `env` object:

```ts aero.config.ts
import { defineConfig } from '@aero-js/core/config'

export default defineConfig(env => ({
	site: env.mode === 'production' ? 'https://example.com' : 'http://localhost:5173',
	content: true,
	server: true,
}))
```

<Note>
	The `env` parameter is the same object Vite passes to a config function — it includes `mode`,
	`command`, and `isSsrBuild`. See the [Vite docs](https://vitejs.dev/config/#conditional-config)
	for details.
</Note>
