When you run `pnpm create @aero-js my-app`, Aero scaffolds a **minimal** tree. The **fullstack** template adds Nitro under `server/` and a root `nitro.config.ts`.

Use the pages below like a directory map: each one describes **what belongs in that folder** and how it maps to URLs or aliases. For how routing, layouts, and templates behave in depth, follow the links into the Guide.

## Minimal template

```
my-app/
├── client/
│   ├── assets/
│   │   ├── scripts/        # Client-side TypeScript/JavaScript
│   │   └── styles/         # CSS files
│   ├── components/         # Reusable .html components
│   ├── layouts/            # Layout wrappers with <slot>
│   └── pages/              # File-based routing
├── content/
│   └── site.ts             # Global site data
├── public/                 # Static assets (copied as-is)
├── vite.config.ts          # Aero Vite plugin
└── tsconfig.json           # Path aliases
```

## Fullstack template

Adds server output and Nitro configuration:

```
my-app/
├── server/
│   └── api/                # Nitro API route handlers
└── nitro.config.ts         # Nitro configuration
```

<Note>
	If you rename top-level folders (for example `frontend/` instead of `client/`), set `dirs` in config
	and read [Path aliases](/configuration/path-aliases).
</Note>
