---
title: "Site URL"
description: "Set your canonical site URL in Aero config to power sitemaps, canonical links, and Open Graph tags."
---

Setting a canonical site URL in your Aero config gives you an absolute base URL you can use for sitemaps, RSS feeds, canonical `<link>` tags, and Open Graph metadata. Without it, Aero can only generate relative URLs.

## Configuration

<Tabs>
  <Tab title="aero.config.ts">
    ```ts
    import { defineConfig } from '@aero-js/config'

    export default defineConfig({
      content: true,
      server: true,
      site: { url: 'https://example.com' },
    })
    ```
  </Tab>
  <Tab title="vite.config.ts">
    ```ts
    import { defineConfig } from 'vite'
    import { aero } from '@aero-js/vite'

    export default defineConfig({
      plugins: [
        aero({
          site: { url: 'https://example.com' },
        }),
      ],
    })
    ```
  </Tab>
</Tabs>

If you omit `site`, the URL defaults to an empty string. Aero won't generate a sitemap and `Aero.site.url` will be falsy.

## Accessing the URL

Once configured, the site URL is available in two places:

| Context                    | How to use it                  |
| -------------------------- | ------------------------------ |
| Build scripts and templates | `Aero.site.url`               |
| Client scripts (Vite)       | `import.meta.env.SITE`        |

### In a layout

Use `Aero.site.url` to build canonical and Open Graph URLs:

```html
<script is:build>
  const base = Aero.site.url || ''
</script>

<link rel="canonical" href="{ base }{ Aero.page.url.pathname }" />
<meta property="og:url" content="{ Aero.site.url }{ Aero.page.url.pathname }" />
```

### In a client script

`import.meta.env.SITE` is replaced at build time by the Aero Vite plugin — no `.env` entry needed:

```html
<script>
  const siteUrl = import.meta.env.SITE
  console.log('Running on:', siteUrl)
</script>
```

## Sitemap generation

When `site.url` is set, Aero automatically generates `sitemap.xml` in your output directory (`dist/`) during a static build. It includes all pre-rendered routes as absolute URLs and excludes the 404 page.

To help crawlers discover it, link to the sitemap from your layout's `<head>`:

```html
<script is:build>
  const base = Aero.site.url || ''
</script>

<link rel="sitemap" type="application/xml" href="{ base }/sitemap.xml" />
```

If you want to omit the link when `site.url` isn't set (e.g. in local dev), use the `if` directive:

```html
<link
  rel="sitemap"
  type="application/xml"
  href="{ Aero.site.url }/sitemap.xml"
  if="{ Aero.site.url }" />
```

<Note>
  The `site` global from your content module (e.g. `content/site.ts`, imported as `@content/site`) is separate from the config `site`. Your content's `site` holds things like page titles and navigation data. The config `site` is only the canonical origin URL.
</Note>
