## Minimal template

## Directory overview

| Directory                | Purpose                                                                                                   |
| ------------------------ | --------------------------------------------------------------------------------------------------------- |
| `client/pages/`          | File-based routing — each `.html` file becomes a URL. See [Routing](/getting-started/routing).            |
| `client/layouts/`        | Shared page shells with `<slot />`. See [Templates](/getting-started/templates).                          |
| `client/components/`     | Reusable `.html` partials. See [Templates](/getting-started/templates).                                   |
| `client/assets/styles/`  | CSS files, referenced via `@styles/` alias. See [Styling](/getting-started/styling).                      |
| `client/assets/scripts/` | Client-side TS/JS, referenced via `@scripts/` alias. See [Scripts](/getting-started/scripts).             |
| `content/`               | Data modules and content collections. See [Content](/getting-started/content).                            |
| `public/`                | Static files copied to `dist/` unchanged (favicons, `robots.txt`). See [Assets](/getting-started/assets). |
| `server/`                | Nitro API handlers (fullstack only). See [Server](/getting-started/server).                               |

## Config files

| File                | Purpose                                                    |
| ------------------- | ---------------------------------------------------------- |
| `vite.config.ts`    | Vite config with the `aero()` plugin                       |
| `aero.config.ts`    | Optional dedicated Aero config via `defineConfig()`        |
| `tsconfig.json`     | TypeScript settings and path aliases                       |
| `nitro.config.ts`   | Nitro config (fullstack only)                              |
| `content.config.ts` | Content collection schemas (when using `@aero-js/content`) |

<Note>
	If you rename top-level folders (for example `frontend/` instead of `client/`), set `dirs` in your
	Aero config. See [Configuration](/getting-started/configuration).
</Note>
