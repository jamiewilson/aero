Loading state

Three independent knobs — easy to mix up because they all relate to “busy” UX.

1. autoDisable — hypermedia sets disabled on the trigger during mutating requests (POST by default). No signal, no busy.

Not saved yet.

2. busy vs disabled — busy wires hypermedia to a signal; it does not change how the button looks. A reactive disabled binding mirrors the signal to the native attribute. Click each — same GET, same state, different attribute.

isSaving: { isSaving ? 'true (in flight)' : 'false (idle)' } — updates for both; only the right button grays out.

3. state shares the signal elsewhere. state: isSaving in the action toggles the binding while the request runs. Other elements can react — status text, sibling buttons — without touching the trigger.
