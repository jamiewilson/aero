Error pages

One template at client/pages/error.html drives every HTTP error response. Aero injects Aero.error (status, message) and keeps Aero.page for request context. The template is not a public route.

404 in dev SSR

Missing pages render error.html with live Aero.page.url.pathname.

  • Open a missing top-level route
  • Open a missing nested route
  • Try to browse /error (blocked — same 404 UX)

Static build artifacts

After pnpm build, Aero prerenders the same template twice with default Aero.error values. Pathname is empty on these files, so copy should not reference the request URL.

  • View dist/404.html
  • View dist/500.html

Server errors (Nitro)

These need the Nitro server (pnpm preview:api or deploy). Dev SSR treats missing pages as 404; thrown routes go through Aero's default errorHandler.

  • Unmatched route (404 via catch-all)
  • Route that throws (500 via error handler)

API errors (JSON)

Under /api/*, the default handler returns JSON — even when Accept prefers HTML.
